### [tools/pi-github](./tools/pi-github)

Multi-platform Git forge automation ‚Äî interact with GitHub, Gitea, and Forgejo APIs through pi, with configurable base URL and authorization for self-hosted instances. Supports **multiple instances** identified by name, so an agent can operate across different platforms and accounts simultaneously.

**Commands:**

- `/gh-login` ‚Äî Add or update a platform instance (GitHub, Gitea, or Forgejo)
- `/gh-default` ‚Äî Set the default platform instance for subsequent tool calls
- `/gh-forget` ‚Äî Remove a configured platform instance
- `/gh-status` ‚Äî Show all configured platform instances

**Tools:**

- `gh_issue_create` ‚Äî Create a new issue on a Git repository. Use the optional 'instance' parameter to target a specific platform (e.g. 'github', 'gitea'). Configure instances via /gh-login.
- `gh_issue_list` ‚Äî List issues from a Git repository with optional filters (state, labels). Use the optional 'instance' parameter to target a specific platform.
- `gh_issue_get` ‚Äî Get detailed information about a specific issue. Use the optional 'instance' parameter to target a specific platform.
- `gh_issue_comment` ‚Äî Add a comment to an existing issue. Use the optional 'instance' parameter to target a specific platform.
- `gh_pr_create` ‚Äî Create a new pull request. Use the optional 'instance' parameter to target a specific platform.
- `gh_pr_list` ‚Äî List pull requests from a Git repository with optional filters. Use the optional 'instance' parameter to target a specific platform.
- `gh_pr_get` ‚Äî Get detailed information about a specific pull request. Use the optional 'instance' parameter to target a specific platform.
- `gh_repo_get` ‚Äî Get information about a Git repository. Use the optional 'instance' parameter to target a specific platform.
